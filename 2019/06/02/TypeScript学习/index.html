Any

any表示可以是任意类型

当我们规定一个数值可以包含所有类型的值时,可以使用any类型

现在，这个数值可以随意添加元素

Void

它表示没有任何类型，通常，当一个函数没有返回值时，我们将一个这个返回值类型定义为void

而将一个变量规定为void类型则没有什么意义，因为你只能为它赋予undefined和null

Null和undefined

这两个类型只能赋值自身类型的唯一值

undefinde和null值是可以赋值给任何类型的变量，这是极其不好的。

官方建议我们指定了--strictNullChecks标记，null和undefined就只能赋值给void和它们各自的类型。

但是–strictNullChecks`标记默认是关闭的，所以我们需要开启它

在根目录下新建一个tsconfig.json，TypeScript 将会把此目录下（和子目录）的所有 .ts 文件做为编译上下文的一部分，它还会包含一部分默认的编译选项。我们就是在json文件在修改这些编译选项

现在，我们返回ts文件，可以看到出来它们自身和void类型被辅助undefined和null，其它的类型都抛出了错误
